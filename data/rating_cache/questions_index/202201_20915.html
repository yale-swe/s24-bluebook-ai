\n \n Navigate to all answers for Respondent Id 1568228\n 1568228\n \n 1568228\n cryptography
\n \n Navigate to all answers for Respondent Id 1572741\n 1572741\n \n 1572741\n We learned about several encryption schemes and definitions of security for them (building up to CPA security and CCA security). We learned about the cryptographic primitive of a One-Way-Function, and how these can be used to build Pseudorandom Generators, which can in turn be used to build Pseudorandom Functions and Pseudorandom Permutations. The latter two can be used to create CPA-secure and CCA-secure encryption schemes, respectively. We studied these extensively in the context of Private-Key Encryption, and we also moved on to Public-Key Encryption. We studied Message Authentication Codes and Digital Signatures. Throughout the course we wrote proofs of security. These proofs of security typically relied on technical assumptions such as FACTORING, DLOG, DDH, or RSA. We also often saw examples of insecure schemes and protocols, and discussed how an adversary can break them. In the last few lectures we discussed some advanced topics such as post-quantum cryptography and cryptocurrencies.
