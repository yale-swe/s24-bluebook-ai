\n \n Navigate to all answers for Respondent Id 1562825\n 1562825\n \n 1562825\n Strengths: The lectures are great and clearly communicate concepts. I don't think I ever felt confused after a lecture and always had a clear understanding of the concepts as they were being taught. I like that Dr. Bennett types up thorough lecture notes which really help when you want to go back and review something. The problem sets are also really interesting applications of the programming languages that you're taught. This class should be listed as one of the options for an introductory programming experience on the computer science major page because it's better than any of the other introductory courses at Yale except maybe CS50 (the quality of each class varies depending on what you're looking for). \r\n\r\nWeaknesses: There aren't really any weaknesses for this course. The exams are a bit of a time crunch but still doable. The grading on the problem sets can be a bit harsh (which is probably due to the fact that there are some overqualified people taking this course), and I think the rubrics should focus more on fundamentals in programming rather than subjective errors like "not having enough comments", but it's not really that big of a deal (just remember to double check that you've stayed consistent with the "good coding style" tips that Dr. Bennett puts in her lecture slides). My only real criticism of the course is that Qt has to be the worst IDE I've ever used (sorry if Qt holds sentimental value to anyone reading this). It's just not friendly with Macs and it's not visually appealing in my opinion). I think the course should simply cut the computer graphics component covered in C and C++ (which is the only reason why we really use Qt), lengthen the time spent on MATLAB to supplement this, and move the coverage of C and C++ to VS Code or GitHub Codespaces. I'm yet to meet somebody that uses C and C++ to implement graphics into code, so I don't think it's an integral component to the coverage of C and C++ in this course and moving to VS Code would make things so much easier in my opinion. If this is, in fact, your first introduction to programming, you probably won't notice that Qt is not that great, but if you've been spoiled with good-looking IDEs like me in the past, just be prepared. Despite this, it's still a great course, and if Dr. Bennett continues to use Qt, you'll live.
