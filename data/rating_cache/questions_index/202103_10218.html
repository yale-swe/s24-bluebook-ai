\n \n Navigate to all answers for Respondent Id 1538645\n 1538645\n \n 1538645\n Strengths: It was a good idea for Professor Karrell to provide r markdown files of the lectures before each class. Professor Karrell was also flexible and generally understanding and funny.\r\n\r\nWeaknesses: It was a bad idea for Professor Karrell to assign a 3rd problem set, a statistical replication project, and a take home final exam to be due all within 1.5 weeks of each other. Because of this, the end of the semester was very stressful. Professor Karrell also needs to be better at editing his slides. He always had typos in his slides and when someone would ask a question, he would blame it on the professor who he inherited the slides from. He also had typos in his assignments. This made it difficult to build confidence in my ability to do statistics because I was never really sure if I was encountering a typo or if I just didn't know what I was doing. For the statistics replication project, he should consider providing mini-assignments along the way to help alleviate the end of semester rush. Maybe the first assignment can be selecting an article, then the next assignment can be data cleaning, and then so forth. Also, he said that learning r would not be the focus of this class. If that's the case, then the course should be modified to lessen the importance of being able to code in r. If you didn't know how to code for this class you most likely failed because everything we did hinged on being able to code in r. There was not enough emphasis on the actual concepts. Something else he could do if he wants to keep things the same is to provide the code we need for everything so that we are just focusing on demonstrating our understanding of the concepts, instead of demonstrating our ability to code in r. Lastly, if coding in r really is that important, maybe he can do a "flipped classroom" type of thing where he does mini-lectures on how to code and posts them to Canvas. That way he doesn't have to spend class time doing it and he doesn't have to field endless questions via email.

\n \n Navigate to all answers for Respondent Id 1549490\n 1549490\n \n 1549490\n I really want to recommend this course because of Professor Karell. My biggest advice to you if you take this course is do not freak out--it's going to be very difficult at first. This class is an introductory course in mathods, which also makes it an introductory R course. A lot of people in it had never even used R, so you can imagine how the first problem set went. Looking back, I think a lot of us worried much more than was merited; you get thrown in the deep end a bit at first, but when you do learn what you're doing it feels so rewarding. Professor Karell is very receptive to student feedback and tweaked the course format so he would have more time in class to help us learn basic R code, and the TAs always answered problem set questions and worked on R concepts in section. As long as you go to lectures, go to section, and always attempt the problem sets so you have some specific points you can ask for help on, you will do alright in this class. One more note: the end of the semester was a bit challenging if you didn't plan out your final project beforehand, as there's also a final exam. They introduce the final project in the first half of the semester, so if you start casual work on it then and you'll be done probably before the last class period and have more time to ask for help!
