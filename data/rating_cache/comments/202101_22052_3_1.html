Q:

Would you recommend this course to another student? Please explain.

 If a student wants to learn "how to code," this is not the course for them -- CS 50 teaches that and much more. However, this is a solid course and thorough introduction to the inner workings of today's modern technology. It truly feels as though one learns of the science behind computers.
 I would only recommend this course if you are planning on majoring in CS. If you are just looking to take a QR credit, I believe there are much easier credits out there. However, the course and homeworks are quite fun once you solve them.
 Yes if you need a CS requirement
 Yes, only if you enjoy logical puzzles. Every Pset in Racket feels like putting together pieces of a puzzle, as many Psets involve coding an implementation of a simulator for a certain construct (turing machine, assembly language simulator, etc) through smaller procedures and then working with that simulator. Additionally, the language is quite non-intuitive at first and codes quite differently from other languages, so you have to be willing to endure the steep learning curve.
 Depends, if you want to get a solid grasp of the some more basic CS concepts, you might want to take the course. If you're a CS major, then you might have to (unless you can skip to 223). Otherwise, I wouldn't recommend it.

 Yes, a great intellectual challenge. What's cool about CS in general is that you may not come up with the same solution as others, but it'll still work!
